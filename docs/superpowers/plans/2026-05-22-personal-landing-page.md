# Personal Landing Page Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a small Astro personal landing page for `rwjblue.com`, with tooling, CI, and Cloudflare deployment documentation.

**Architecture:** The site is a static Astro app with one page and one stylesheet. Tool versions are pinned through mise, local workflows are exposed through file-based mise tasks, and deployment is documented for Cloudflare Pages serving the same build from multiple domains without redirects.

**Tech Stack:** Astro, npm, mise, GitHub Actions, Cloudflare Pages.

---

## File Structure

- `.gitignore`: ignores dependencies, build output, local env files, and `.superpowers/` scratch data.
- `.env.example`: documents optional Cloudflare deployment variables without real secrets.
- `.mise/config.toml`: pins Node LTS and loads local `.env`.
- `.mise/mise.lock`: generated by `mise install` and committed.
- `.mise/tasks/install`: file-based task wrapping `npm install`.
- `.mise/tasks/dev`: file-based task wrapping `npm run dev`.
- `.mise/tasks/check`: file-based task wrapping `npm run check`.
- `.mise/tasks/build`: file-based task wrapping `npm run build`.
- `.mise/tasks/preview`: file-based task wrapping `npm run preview`.
- `.github/workflows/ci.yml`: validates install, Astro sync, and production build.
- `package.json`: npm metadata, scripts, and dev dependencies.
- `package-lock.json`: generated by `npm install`.
- `astro.config.mjs`: minimal Astro config for a static site.
- `src/pages/index.astro`: single landing page.
- `src/styles/global.css`: all page styling.
- `public/favicon.svg`: small static favicon.
- `docs/deployment.md`: Cloudflare Pages setup and domain strategy.
- `README.md`: project overview and local workflow.

### Task 4: GitHub Actions CI

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: Create CI workflow**

Create `.github/workflows/ci.yml`:

```yaml
name: CI

on:
  pull_request:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Check out repository
        uses: actions/checkout@v4

      - name: Install mise
        uses: jdx/mise-action@v2

      - name: Install project dependencies
        run: mise run install

      - name: Sync Astro types
        run: mise run check

      - name: Build site
        run: mise run build
```
